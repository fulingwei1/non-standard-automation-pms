# Role Management Frontend UAT Checklist

> 生成时间：2026-03-26
> 目标：验证“角色管理专项”前端功能是否**真正可用**，而不是只有后台接口或前端展示
> 适用范围：角色模板、复制角色、权限矩阵、效果预览/模拟器、风险提示、导入导出、批量赋角色

---

## 一、验收原则

本清单只看 3 件事：
1. **前端是否真的可见、可操作**
2. **前端是否真的调用后端并写回成功**
3. **前端展示结果是否与后端/权限语义一致**

### 验收结论分类
- ✅ **通过**：前端可操作 + 后端写回成功 + 刷新后结果一致
- ⚠️ **部分通过**：前端可见，但仅展示或部分链路未打通
- ❌ **失败**：前端入口缺失、点击无效、写回失败、结果不一致

---

## 二、验收准备

### 环境准备
- 前端启动
- 后端启动
- 使用具有权限管理权限的账号登录
- 准备至少 3 个可测试角色：
  - 普通角色（如 SALES）
  - 管理角色（如 SALES_DIR / GM）
  - 风险较高角色（权限较多）

### 建议准备测试用户
- 用户 A / B / C
- 至少一个用户没有角色
- 至少一个用户已有多个角色

## 三、角色管理前端总入口检查

### 3.1 页面入口
- [ ] 能进入角色管理页面
- [ ] 页面无白屏/报错
- [ ] 角色列表能正常展示
- [ ] 操作列能看到：
  - [ ] 编辑
  - [ ] 复制角色
  - [ ] 保存为模板（如紫色图标）
  - [ ] 其他扩展操作

### 3.2 角色编辑基础体验
- [ ] 打开角色编辑对话框无异常
- [ ] 默认进入 **预览/效果** 相关 tab（不是 basic）
- [ ] 仍可手动切回“基本信息”/“权限管理”等 tab
- [ ] 保存按钮可见且不因切换 tab 消失

## 四、复制角色验收

## 4.1 UI 可见性
- [ ] 角色列表中存在“复制角色”入口
- [ ] 点击后弹出复制角色对话框
- [ ] 对话框中自动带出源角色信息
- [ ] 能看到复制选项说明：
  - [ ] 复制权限
  - [ ] 复制数据范围
  - [ ] 复制导航/菜单（若有）

## 4.2 表单校验
- [ ] 角色编码必须以字母开头、下划线规范通过
- [ ] 编码为空时有提示
- [ ] 角色名称为空时有提示
- [ ] 输入已存在角色编码时，能看到明确错误，而不是只有 alert
- [ ] 输入保留编码（如 ADMIN）时能被拦截

## 4.3 提交链路
- [ ] 提交后请求成功
- [ ] 新角色出现在角色列表中
- [ ] 打开新角色详情，权限/数据范围与开关设置一致
- [ ] 关闭“复制权限”后，新角色权限为空或按预期减少
- [ ] 关闭“复制数据范围”后，默认降级为更安全范围（如 OWN / basic）

## 4.4 API 验证（可选）
- [ ] `POST /api/v1/roles/{id}/copy` 正常返回

### 结论
- [ ] 复制角色：✅ / ⚠️ / ❌

## 五、模板中心验收

## 5.1 模板入口
- [ ] 页面中存在“模板中心”或等价入口
- [ ] 可查看模板列表
- [ ] 模板列表显示关键元信息：
  - [ ] 模板名
  - [ ] 模板编码
  - [ ] 来源角色/说明
  - [ ] 版本（如有）
  - [ ] 启用状态（如有）

## 5.2 角色另存为模板
- [ ] 在角色列表中有“保存为模板”入口
- [ ] 点击后弹出保存模板对话框
- [ ] 名称/编码默认值合理
- [ ] 同编码重复时有明确提示
- [ ] 保存成功后，模板中心可看到新模板

## 5.3 从模板创建角色
- [ ] 存在“从模板创建角色”入口
- [ ] 选择模板后可自动带出默认内容
- [ ] 创建成功后，角色列表中能看到新角色
- [ ] 新角色 data_scope / 权限与模板一致

## 5.4 模板版本
- [ ] 模板更新后版本号递增（如已实现）
- [ ] 仅切换启用状态时版本是否保持不变（如设计如此）

## 5.5 API 验证（可选）
- [ ] `GET /api/v1/roles/templates`
- [ ] `POST /api/v1/roles/templates/{id}/create-role`
- [ ] `POST /api/v1/roles/{id}/save-as-template`

### 结论
- [ ] 模板中心：✅ / ⚠️ / ❌

## 六、权限矩阵验收

## 6.1 视图切换
- [ ] 权限管理 tab 中存在 **矩阵 / 列表** 切换
- [ ] 切换后内容正常显示
- [ ] 不会丢失已勾选状态

## 6.2 模块化展示
- [ ] 至少能看到这些模块分组：
  - [ ] 销售
  - [ ] 审批
  - [ ] 仓储
  - [ ] 工程绩效
  - [ ] 系统管理
- [ ] 每个模块下操作分组清晰
- [ ] 高风险操作有明显标记：
  - [ ] APPROVE
  - [ ] EXPORT
  - [ ] MANAGE
  - [ ] CONFIG

## 6.3 快捷操作
- [ ] 每个模块右侧有：
  - [ ] 只读
  - [ ] 编辑
  - [ ] 管理
  - [ ] 清空
- [ ] 点击“只读”后，该模块仅保留读类权限
- [ ] 点击“编辑”后，读 + 写类权限按预期被勾选
- [ ] 点击“管理”后，高风险管理项按预期被勾选
- [ ] 点击“清空”后，该模块权限清空

## 6.4 写回验证
- [ ] 修改矩阵勾选后点击保存
- [ ] 保存成功
- [ ] 刷新角色详情或重新打开对话框，勾选结果仍然存在
- [ ] 快捷按钮不是只改前端状态，而是后端真的写回

## 6.5 API 验证（可选）
- [ ] `PUT /api/v1/roles/{id}/permissions`

### 结论
- [ ] 权限矩阵：✅ / ⚠️ / ❌

## 七、权限效果预览 / 模拟器验收

## 7.1 基础可见性
- [ ] 打开角色编辑时默认进入预览 tab
- [ ] 预览 tab 不影响保存
- [ ] 预览中能看到模块权限摘要
- [ ] 预览中能看到高风险权限高亮
- [ ] 预览中能看到数据范围摘要

## 7.2 模块差异化表达
- [ ] sales 的范围说明合理（本人/团队/部门/全部）
- [ ] approvals 的说明体现 participant/workflow visibility
- [ ] engineer_performance 的说明体现 data scope 语义

## 7.3 一致性验证
- [ ] 勾选权限前后，预览内容立即变化
- [ ] 继承权限与直接权限合并结果合理
- [ ] 保存后重新打开，预览与实际权限一致

## 7.4 风险提示
- [ ] 勾选多个 DELETE/APPROVE/MANAGE 权限后，出现“过宽角色”提醒
- [ ] 取消某模块 VIEW 权限但保留 DELETE/APPROVE 时，出现“缺失基础权限”提醒
- [ ] 风险提示不是误报，不会在明显安全组合下乱跳

## 7.5 模拟器（如有独立路由）
- [ ] 能进入 `/system/permission-simulator`（若已实现）
- [ ] 选择不同角色/用户后结果会变化
- [ ] sales / approvals / engperf 三类结果可读

### 结论
- [ ] 预览/模拟器：✅ / ⚠️ / ❌

## 9.3 前端可解释性
- [ ] 风险规则解释清楚
- [ ] 管理员知道为什么被判高风险

### 结论
- [ ] 风险评分：✅ / ⚠️ / ❌

## 十一、最终验收建议

### 若以下项全部为 ✅
则可以认定：

# 角色管理前端已真正接入后端，且不是只有后台功能

### 若以下项中有 1~2 个为 ⚠️
建议：
- 仍可主线落地
- 但需标注为 MVP / 第一阶段

### 若存在任意核心项为 ❌
建议：
- 不要再宣称“前端已完成”
- 应先补齐链路，再并主线

核心项包括：
- 复制角色
- 权限矩阵写回
- 预览/模拟器一致性
- 模板中心

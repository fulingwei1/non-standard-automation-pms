## 二、项目阶段与状态

### 2.1 项目生命周期阶段

| 阶段 | 阶段编码 | 阶段名称 | 典型周期 | 主要交付物 | 关键里程碑 |
|------|---------|---------|---------|-----------|-----------|
| 1 | initiation | 立项启动 | 3-5天 | 项目立项书、初步方案 | 立项评审通过 |
| 2 | solution | 方案设计 | 5-10天 | 技术方案、报价单、BOM初稿 | 方案评审通过 |
| 3 | mechanical | 结构设计 | 10-20天 | 3D模型、工程图纸、BOM确认 | 设计评审通过 |
| 4 | electrical | 电气设计 | 7-15天 | 电气图纸、程序框架、接线清单 | 电气评审通过 |
| 5 | procurement | 采购制造 | 15-30天 | 物料到货、外协件加工完成 | 物料齐套 |
| 6 | assembly | 装配调试 | 10-20天 | 设备组装完成、单机调试OK | 厂内验收通过 |
| 7 | acceptance | 验收交付 | 3-7天 | 验收报告、操作手册、培训记录 | 客户签收 |
| 8 | warranty | 质保服务 | 按合同 | 售后服务记录 | 质保期结束 |

### 2.2 项目状态

| 状态 | 状态编码 | 说明 | 允许操作 |
|------|---------|------|---------|
| 未启动 | pending | 项目已创建但未正式启动 | 编辑、启动、取消 |
| 进行中 | active | 项目正在执行 | 编辑、更新进度、暂停、完成 |
| 已暂停 | paused | 项目临时暂停（等待客户、缺料等） | 恢复、取消 |
| 已完成 | completed | 项目已交付验收 | 查看、导出 |
| 已取消 | cancelled | 项目被取消 | 查看 |
| 已归档 | archived | 项目已归档（质保结束） | 查看 |

### 2.3 任务状态

| 状态 | 状态编码 | 说明 | 颜色 |
|------|---------|------|------|
| 未开始 | pending | 任务尚未开始 | 灰色 |
| 进行中 | in_progress | 任务正在执行 | 蓝色 |
| 已完成 | completed | 任务已完成 | 绿色 |
| 阻塞 | blocked | 任务被阻塞（等待前置、缺料等） | 红色 |
| 已取消 | cancelled | 任务被取消 | 灰色 |

## 三、项目健康度

### 3.1 健康度指标

| 指标 | 计算方式 | 绿灯 | 黄灯 | 红灯 |
|------|---------|------|------|------|
| **进度绩效指数(SPI)** | 实际进度 / 计划进度 | ≥0.95 | 0.85-0.95 | <0.85 |
| **成本绩效指数(CPI)** | 预算成本 / 实际成本 | ≥0.95 | 0.85-0.95 | <0.85 |
| **任务逾期率** | 逾期任务数 / 总任务数 | ≤5% | 5%-15% | >15% |
| **里程碑达成率** | 按时完成里程碑 / 总里程碑 | ≥90% | 70%-90% | <70% |

### 3.2 综合健康状态

| 状态 | 颜色 | 判定条件 | 处理建议 |
|------|------|---------|---------|
| 正常 | 🟢 绿色 | 所有指标均为绿灯 | 按计划执行 |
| 关注 | 🟡 黄色 | 任一指标为黄灯 | 关注风险，制定预案 |
| 预警 | 🔴 红色 | 任一指标为红灯 | 立即介入，采取纠偏措施 |

### 3.3 预警类型

| 预警类型 | 预警编码 | 触发条件 | 预警级别 | 通知对象 |
|---------|---------|---------|---------|---------|
| 任务逾期 | task_overdue | 任务超过计划结束日期 | 黄→橙→红 | 负责人、PM、组长 |
| 进度滞后 | progress_delay | SPI < 0.9 | 黄→红 | PM、TE、部门经理 |
| 任务即将到期 | task_due_soon | 3天内到期且进度<80% | 黄 | 负责人 |
| 里程碑风险 | milestone_risk | 里程碑7天内到期且前置未完成 | 橙 | PM、TE |
| 负荷过高 | workload_high | 人员负荷率>120% | 黄→橙 | 组长、部门经理 |
| 成本超支 | cost_overrun | 实际成本>预算×110% | 橙→红 | PM、财务 |
| 物料延期 | material_delay | 物料未按期到货 | 黄→红 | 采购、PM |
| 质量问题 | quality_issue | 发现重大质量问题 | 红 | QA、TE、PM |

## 六、典型工作流

### 6.1 项目立项流程

```
销售提交需求 → PM初评 → TE可行性分析 → 部门经理审核 → 总经理审批 → 立项成功
                                                    ↓
                                              驳回修改
```

### 6.2 任务分配流程

```
PM创建WBS → TE细化任务 → 组长分配到人 → 工程师确认接收 → 开始执行
                                            ↓
                                       反馈工作量问题
```

### 6.3 进度更新流程

```
工程师更新任务进度 → 系统自动计算父任务进度 → 系统计算项目进度 → 触发健康度检查
                                                              ↓
                                                    健康度异常 → 触发预警 → 通知相关人员
```

### 6.4 工时审批流程

```
工程师填报工时 → 提交审批 → 组长审批 → PM确认 → 工时生效
                            ↓           ↓
                          驳回        驳回
```

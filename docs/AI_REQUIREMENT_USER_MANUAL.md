# AI智能需求理解引擎 - 用户使用手册

## 📘 产品简介

AI智能需求理解引擎是一款专为非标自动化项目设计的智能需求分析工具。它能够**自动理解客户的自然语言需求**，快速生成结构化的技术规格文档，大幅提升售前工作效率。

### 核心价值

- ⚡ **效率提升**: 将2-3小时的需求分析工作缩短至3秒内
- 🎯 **准确识别**: AI准确率>85%，自动识别设备、工艺、参数
- 💡 **智能提问**: 自动生成5-10个澄清问题，避免遗漏关键信息
- 📊 **结构化输出**: 标准化的技术规格、设备清单、工艺流程
- 🔍 **可行性评估**: 自动评估技术风险和资源需求

---

## 🚀 快速开始

### 第一步：创建需求分析

1. **登录系统**，进入售前模块
2. **选择或创建**一个售前工单
3. **粘贴客户需求**到"原始需求"输入框
4. **点击"AI分析"**按钮
5. **等待3秒**，查看分析结果

**示例需求**:
```
我们是一家手机制造商，需要一条自动化生产线用于手机外壳的组装和检测。

基本要求：
- 产能：每小时200件
- 工艺流程：上料 -> 视觉检测 -> 自动组装 -> 质量检测 -> 下料
- 设备要求：工业机器人、视觉检测系统、自动组装设备、质检设备
- 精度要求：±0.05mm
- 环境要求：洁净车间，温度20-25℃
- 预算：500万以内
- 工期：6个月
```

**系统会自动输出**:
- ✅ 结构化需求文档
- ✅ 设备清单（含规格建议）
- ✅ 工艺流程图数据
- ✅ 技术参数规格表
- ✅ 5-10个澄清问题
- ✅ 技术可行性分析
- ✅ 置信度评分

## 📋 功能详解

### 1️⃣ 结构化需求文档

AI会将客户的描述转化为标准化的需求文档：

**输出内容**:
- **项目类型**: 手机外壳组装自动化
- **应用行业**: 消费电子
- **核心目标**: 提升产能、保证质量、降低人工成本
- **功能性需求**: 自动上料、视觉检测、自动组装、质量检测、自动下料
- **非功能性需求**: 产能≥200件/小时、精度±0.05mm、洁净车间
- **约束条件**: 温度20-25℃、预算500万、工期6个月
- **假设条件**: 手机外壳标准化、现场具备基础设施

**如何使用**:
1. 查看AI生成的结构化需求
2. 如有遗漏或错误，点击"编辑"手动修正
3. 确认无误后，点击"保存"

1. 检查设备清单的完整性

AI会生成初步的工艺流程设计。

**示例输出**:

```
步骤1: 上料
描述: 自动上料系统将工件放置到传送带
参数: 节拍时间18秒/件
设备: 上料机构、传送带

步骤2: 视觉检测
描述: 3D视觉系统检测工件外观和尺寸
参数: 检测时间3秒/件，精度±0.01mm
设备: 3D视觉系统

步骤3: 自动组装
描述: 机器人执行自动组装
参数: 组装时间12秒/件
设备: 六轴工业机器人、力控传感器

步骤4: 质量检测
描述: 检测组装质量
参数: 检测时间5秒/件
设备: 视觉检测系统、力测试仪

步骤5: 下料
描述: 合格品自动下料，不合格品剔除
参数: 节拍时间3秒/件
```

**如何使用**:
1. 审查工艺流程的合理性
2. 调整步骤顺序
3. 补充遗漏步骤
4. 细化工艺参数

### 5️⃣ 澄清问题（智能提问）

AI会自动生成5-10个澄清问题，帮助你获取更多细节。

**示例输出**:

**🔴 关键问题（必须澄清）**:
1. **[技术参数]** 请明确视觉检测系统的分辨率和检测速度要求
2. **[验收标准]** 项目的验收标准和成功指标是什么？

**🟠 高优先级问题**:
3. **[功能需求]** 组装过程是否需要力控功能？
   - *建议*: 建议配置力控传感器以保证组装质量
4. **[约束条件]** 现场是否有特殊的空间、环境或安全约束？
5. **[技术参数]** 换型频率和换型时间要求是什么？

**🟡 中优先级问题**:
6. **[功能需求]** 是否需要数据采集和MES系统对接？
   - *建议*: 建议集成MES接口以实现生产数据实时监控
7. **[资源预算]** 项目的预算范围和交付时间要求是什么？

**如何使用**:
1. 将问题发送给客户
2. 根据客户回答，使用"精炼需求"功能
3. 重复分析，提升置信度

### 6️⃣ 技术可行性分析

AI会评估项目的技术可行性和风险。

**示例输出**:

**总体可行性**: 🟢 高 (High)

**技术风险**:
- ⚠️ 视觉检测算法需要针对性训练
- ⚠️ 多工位协调同步需要精密调试

**资源需求**:
- 人力: 3名集成工程师，2名调试工程师
- 设备: 六轴机器人×2，3D视觉系统×2
- 周期: 设计2个月，集成调试3个月

**复杂度评估**: 🟡 中等 (Medium)

**开发挑战**:
- 多工位协调同步
- 视觉算法优化
- 快速换型机构设计

**建议**:
- 建议先进行工艺可行性验证
- 考虑分阶段实施
- 建议预留10%的技术储备时间

**如何使用**:
1. 根据可行性评估决定是否接单
2. 识别关键风险点
3. 制定风险应对策略
4. 合理规划资源和时间

### 7️⃣ 置信度评分

系统会给出AI理解需求的置信度评分，帮助你判断需求的清晰程度。

**评分示例**:

```
总体置信度: 82%  🟢

评分分解:
  需求完整性: 25/30  ████████░░
  技术清晰度: 24/30  ████████░░
  参数明确性: 21/25  ████████░░
  可执行性:   12/15  ████████░░

评估: 中等置信度
建议:
  ✅ 需求基本清晰，建议澄清部分细节
  ⚠️ 重点关注技术参数的明确性
  💡 建议补充验收标准
```

**置信度等级**:

| 分数 | 等级 | 说明 | 建议操作 |
|------|------|------|----------|
| ≥85% | 🟢 高 | 需求理解充分 | 可以进入方案设计阶段 |
| 60-84% | 🟡 中 | 需求基本清晰 | 建议澄清部分细节 |
| <60% | 🔴 低 | 需求信息不足 | 需要与客户进一步沟通 |

**如何使用**:
1. 查看置信度评分
2. 如果<85%，查看"建议"
3. 使用"澄清问题"功能获取更多信息
4. 使用"精炼需求"功能提升置信度

## 💡 使用技巧

### 技巧1: 如何撰写好的需求描述

**推荐格式**:
```
项目背景：简述项目目的和应用场景

核心要求：
- 产能/产量要求
- 质量/精度要求
- 工艺流程描述

技术参数：
- 关键设备要求
- 性能指标
- 环境条件

约束条件：
- 预算范围
- 交付时间
- 现场限制

验收标准：
- 成功指标
- 质量标准
```

**✅ 好的示例**:
```
我们需要一条锂电池自动化组装线。
产能：500件/天
工艺：极耳焊接 -> 电解液注液 -> 封装 -> 测试
精度要求：焊接精度±0.02mm，注液精度±5ml
环境：防爆车间，湿度<50%
预算：300万，工期4个月
```

**❌ 不好的示例**:
```
需要一条自动化生产线，帮我报个价。
```

### 技巧2: 如何提升置信度

1. **补充详细信息**: 技术参数、性能指标、环境条件
2. **明确约束条件**: 预算、时间、空间限制
3. **描述工艺流程**: 尽量详细描述每个步骤
4. **说明验收标准**: 明确成功指标

### 技巧3: 如何使用澄清问题

1. **按优先级处理**: 先解决"关键问题"，再处理"高优先级问题"
2. **利用建议答案**: 参考AI提供的建议答案
3. **整理客户回答**: 使用"精炼需求"功能输入客户的回答
4. **持续迭代**: 多次精炼，直到置信度≥85%

### 技巧4: 如何手动修正AI错误

AI有时会识别错误，你可以：

1. **修正设备清单**: 删除错误设备，添加遗漏设备
2. **调整工艺流程**: 修改步骤顺序，补充缺失步骤
3. **更新技术参数**: 修正参数值，添加新参数
4. **完善验收标准**: 根据实际情况调整标准

## ❓ 常见问题 (FAQ)

### Q1: AI分析失败怎么办？

**A**: 可能原因和解决方案：
1. **网络问题**: 检查网络连接，重试
2. **API配额用尽**: 联系系统管理员
3. **需求描述太短**: 确保需求描述≥10个字符
4. **服务器繁忙**: 稍后重试

如果问题持续，联系技术支持。

### Q2: 置信度一直很低怎么办？

**A**: 
1. 查看"建议"，了解问题所在
2. 与客户沟通，获取更多详细信息
3. 使用"精炼需求"功能，多次迭代
4. 如果仍然<60%，建议人工介入

### Q3: AI识别的设备不准确怎么办？

**A**:
1. 点击"编辑设备清单"
2. 删除错误的设备
3. 手动添加正确的设备
4. 调整规格参数
5. 保存更新

### Q4: 澄清问题太多，如何筛选？

**A**:
1. 优先处理"关键问题"（红色标记）
2. 再处理"高优先级问题"（橙色标记）
3. 中低优先级问题可根据实际情况选择性处理
4. 通常5-8个问题即可

### Q5: 如何导出分析报告？

**A**:
1. 在分析结果页面，点击"导出报告"
2. 选择格式：PDF / Word / Excel
3. 选择包含的内容（全部或部分）
4. 点击"下载"

### Q6: 可以批量分析多个需求吗？

**A**: 
目前不支持批量分析。建议：
1. 逐个分析每个需求
2. 使用"模板"功能复用相似需求
3. 联系技术支持了解批量分析功能的开发计划

# 会议报告自动生成使用指南

## 一、功能概述

系统支持自动生成**年度会议报告**和**月度会议报告**，月度报告自动包含与上月的对比数据，帮助管理层快速了解会议执行情况和趋势变化。

## 二、报告类型

### 2.1 年度会议报告

**功能**：
- 汇总指定年份的所有会议数据
- 统计会议数量、完成情况、行动项执行情况
- 收集关键决策和战略结构
- 按层级统计会议分布

**适用场景**：
- 年度战略复盘
- 年度工作总结
- 管理层汇报

### 2.2 月度会议报告

**功能**：
- 汇总指定月份的所有会议数据
- **自动与上月对比**，显示变化趋势
- 统计会议数量、完成情况、行动项执行情况
- 收集关键决策

**对比维度**：
- 会议总数对比
- 已完成会议数对比
- 行动项总数对比
- 已完成行动项数对比
- 完成率对比

**适用场景**：
- 月度经营分析会
- 月度工作总结
- 趋势分析

## 三、使用流程

### 3.1 生成报告

1. **进入报告页面**
   - 访问路径：`/meeting-reports`
   - 或从管理节律仪表盘导航

2. **点击"生成报告"按钮**

3. **填写生成参数**：
   - **报告类型**：选择"年度报告"或"月度报告"
   - **年份**：选择报告年份
   - **月份**：月度报告必填（1-12月）
   - **节律层级**：可选，筛选特定层级的会议（如仅统计战略层）

4. **点击"生成报告"**
   - 系统自动汇总数据
   - 月度报告自动计算与上月对比
   - 生成报告记录

### 3.2 查看报告

1. **报告列表**
   - 显示所有已生成的报告
   - 支持按类型、年份、层级筛选
   - 显示报告摘要信息

2. **报告详情**
   - 点击"查看"按钮
   - 查看完整的报告数据
   - 查看对比数据（月度报告）

### 3.3 报告内容

**报告摘要**：
- 会议总数
- 已完成会议数
- 行动项总数
- 已完成行动项数
- 完成率

**详细数据**：
- 会议列表（包含会议名称、日期、状态、行动项数量）
- 行动项统计（总数、已完成、逾期、进行中）
- 关键决策列表
- 战略结构（年度报告）
- 按层级统计

**对比数据**（仅月度报告）：
- 与上月会议数对比（变化量、变化率）
- 与上月行动项对比
- 与上月完成率对比
- 趋势指示（上升/下降/持平）

## 四、报告数据说明

### 4.1 报告编号规则

- **年度报告**：`MR-{年份}-ANNUAL-{时间戳}`
- **月度报告**：`MR-{年份}{月份}-MONTHLY-{时间戳}`

示例：
- `MR-2025-ANNUAL-20250109120000`
- `MR-202501-MONTHLY-20250109120000`

### 4.2 对比数据格式

月度报告的对比数据包含：

```json
{
  "previous_period": "2025-01",  // 上月周期
  "current_period": "2025-02",   // 当前周期
  "meetings_comparison": {
    "current": 8,                // 本月会议数
    "previous": 6,                // 上月会议数
    "change": 2,                  // 变化量
    "change_rate": "+33.3%"      // 变化率
  },
  "completion_rate_comparison": {
    "current": "90.0%",          // 本月完成率
    "previous": "85.0%",         // 上月完成率
    "change": "+5.0%"             // 变化
  }
}
```

### 4.3 趋势指示

- **上升趋势**：绿色，显示上升箭头
- **下降趋势**：红色，显示下降箭头
- **持平**：灰色，显示横线

## 五、最佳实践

### 5.1 报告生成时机

**年度报告**：
- 建议在每年12月底或次年1月初生成
- 用于年度战略复盘和总结

**月度报告**：
- 建议在每月1-3日生成上月报告
- 用于月度经营分析会
- 及时了解月度变化趋势

### 5.2 报告使用场景

1. **管理层汇报**
   - 使用年度报告展示全年成果
   - 使用月度报告展示月度进展

2. **趋势分析**
   - 对比连续多个月的月度报告
   - 识别会议执行趋势
   - 发现潜在问题

3. **决策支持**
   - 基于报告数据调整管理策略
   - 优化会议安排
   - 改进行动项管理

### 5.3 报告筛选建议

- **全公司报告**：不选择节律层级，查看所有层级
- **战略层报告**：选择"战略层"，仅统计战略会议
- **经营层报告**：选择"经营层"，仅统计经营分析会

## 六、常见问题

### Q1: 月度报告的上月对比是如何计算的？

A: 系统自动计算：
- 如果当前是2月，则对比1月数据
- 如果当前是1月，则对比去年12月数据
- 对比维度包括：会议数、行动项数、完成率等

### Q2: 报告生成需要多长时间？

A: 通常几秒钟即可完成，取决于：
- 会议数量
- 行动项数量
- 数据复杂度

### Q3: 可以重新生成报告吗？

A: 可以。系统允许为同一周期生成多个报告，每个报告有唯一的报告编号。

### Q4: 报告数据是实时的吗？

A: 报告数据是生成时的快照，不会自动更新。如需最新数据，请重新生成报告。

### Q5: 可以导出报告吗？

A: 报告数据存储在数据库中，可以通过API获取。文件导出功能（PDF/Excel）正在开发中。

## 七、API接口

### 生成报告

```http
POST /api/v1/meeting-reports/generate
Content-Type: application/json

{
  "report_type": "MONTHLY",
  "period_year": 2025,
  "period_month": 1,
  "rhythm_level": "STRATEGIC"
}
```

### 获取报告列表

```http
GET /api/v1/meeting-reports?page=1&page_size=20&report_type=MONTHLY&period_year=2025
```

### 获取报告详情

```http
GET /api/v1/meeting-reports/{report_id}
```

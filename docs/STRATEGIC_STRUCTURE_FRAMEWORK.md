# 五层战略结构框架使用指南

## 一、框架概述

真正的战略是一整套结构：从愿景、机会、定位，到目标、路径，从看见到走通！

本系统将五层战略结构融入管理节律与会议地图系统，帮助企业在战略会上真正"想透、说透、做透"。

## 二、五层结构详解

### 第一层：使命/愿景（Vision/Mission）

**核心问题**：我们要成为什么？

**关键要素**：
- **使命**：我们希望通过____，让____变得更好
- **愿景**：最终成为一家怎样的公司
- **存在的意义**：我们为什么存在？
- **三年后希望被怎么记住**：如果三年后只剩一句话，我们希望被怎么记住？
- **为谁创造长期价值**：我们希望为谁创造长期价值？

**判断标准**：
- 能否回答"存在的意义"？
- 是否具备超越利益的价值追求？
- 是否能激发长期共识而非短期业绩？

**常见误区**：把愿景当战略，以为喊理想就能引方向

### 第二层：战略机会（Opportunity）

**核心问题**：我们凭什么能赢？

**关键要素**：
- **市场趋势**：未来三年的行业关键趋势
- **客户需求本质**：客户最根本的需求是什么？
- **竞争空位**：竞争中的空隙在哪里？
- **我们的优势**：我们凭什么能赢？
- **四点合一分析**：
  - 行业是否有增长？
  - 竞争是否有空隙？
  - 客户是否有需求？
  - 我们是否有优势？

**判断标准**：
- 是否识别到结构性机会？
- 是否明确"凭什么能赢"的逻辑？
- 是否形成可被验证的假设？

**常见误区**：把目标当战略，以为有数字就能控全局

### 第三层：战略定位（Positioning）

**核心问题**：我们在哪个战场打，用什么方式取胜？

**关键要素**：
- **聚焦的赛道/细分市场**：我们要在哪个细分市场建立主阵地？
- **差异化方式**：我们打算通过什么样的差异化来形成势能？
- **核心客户群体**：我们的核心客户是谁？
- **独特价值主张**：我们为客户提供什么独特价值？
- **竞争壁垒**：我们的竞争壁垒是什么？

**判断标准**：
- 是否明确"赛道选择 + 核心打法"？
- 是否有排他性（别人难以复制）？
- 是否与资源和能力相匹配？

**常见误区**：定位不清晰，什么都想做

### 第四层：战略目标（Goals）

**核心问题**：我们要实现什么结果？

**关键要素**：
- **战略假设**：我们的战略假设是什么？
- **关键指标**：用于验证假设的指标（名称、目标值、验证目的）
- **年度目标**：年度目标
- **季度目标**：季度目标
- **月度目标**：月度目标

**判断标准**：
- 是否能用数据验证战略逻辑？
- 是否与经营分析会的节奏衔接？
- 是否形成多层目标树（年度 → 季度 → 月度）？

**常见误区**：设"漂亮数字"，不设"可验证逻辑"

### 第五层：战略路径（Path）

**核心问题**：我们怎么实现？

**关键要素**：
- **客户需求本质**：客户需求本质是什么？
- **产品方案**：我们在客户需求本质上应该提供什么产品？
- **服务体验**：我们在客户需求本质上应该创造什么体验？
- **复购机制**：我们在客户需求本质上如何让客户持续复购？
- **执行次序**：执行的先后顺序是什么？
- **节奏**：节奏(周执行/月验证/季校准/年复盘)
- **兵力投入**：在哪些领域投入主要资源？
- **战役系统**：关键战役和里程碑

**判断标准**：
- 是否打通从客户需求到价值交付的链路？
- 是否有节奏、次序、兵力机制支撑执行？
- 是否能通过经营分析实现动态验证？

**常见误区**：把战略当执行，以为做计划就能见结果

## 三、系统使用

### 3.1 在战略会议中记录五层结构

1. 创建或打开战略会议
2. 进入"五层战略结构"标签页
3. 逐层填写每层的内容
4. 点击"保存结构"保存

### 3.2 五层结构的逻辑关系

```
愿景（为什么存在）
    ↓
机会（凭什么能赢）
    ↓
定位（在哪里打、怎么打）
    ↓
目标（要验证什么）
    ↓
路径（怎么落地）
```

每一层都回答一个根本问题，层层递进，环环相扣。

### 3.3 战略会的正确开法

**错误做法**：
- 把战略会开成目标会
- 只讨论数字，不讨论逻辑
- 把愿景当战略
- 把执行当判断

**正确做法**：
1. **先想透**：逐层推演五层结构
2. **再说透**：用结构化的方式表达战略
3. **再做透**：通过路径层落地执行

### 3.4 与经营分析会的衔接

- **战略会**：讨论五层结构，确定战略方向
- **经营分析会**：验证战略目标，分析差距，调整路径
- **运营例会**：执行战略路径，监控关键指标
- **日清会**：完成具体任务，清除障碍

## 四、最佳实践

### 4.1 战略会准备

1. **提前准备**：会议前1-2周准备五层结构初稿
2. **数据支撑**：准备市场数据、竞争分析、客户调研
3. **明确目标**：明确本次会议要讨论哪几层

### 4.2 战略会执行

1. **逐层讨论**：按照五层顺序逐层讨论
2. **逻辑验证**：每层都要验证逻辑是否自洽
3. **记录决策**：及时记录关键决策和假设

### 4.3 战略会跟进

1. **结构落地**：将五层结构转化为具体行动项
2. **定期验证**：通过经营分析会验证战略假设
3. **动态调整**：根据验证结果调整定位、目标或路径

## 五、常见问题

### Q1: 五层结构必须全部填写吗？

A: 不是。可以根据会议类型和阶段选择重点讨论的层级。例如：
- 年度战略会：重点讨论愿景、机会、定位
- 季度战略复盘会：重点讨论目标、路径
- 月度战略解码会：重点讨论目标拆解

### Q2: 如何判断五层结构是否完整？

A: 检查以下几点：
- 每层是否回答了核心问题？
- 层与层之间是否有逻辑关系？
- 是否可以从愿景推演到路径？

### Q3: 五层结构可以修改吗？

A: 可以。战略是动态的，应该根据市场变化和验证结果进行调整。但修改要有依据，不能随意变更。

### Q4: 如何避免把战略会开成目标会？

A: 确保会议讨论顺序：
1. 先讨论愿景和机会（为什么做、凭什么赢）
2. 再讨论定位（在哪里打、怎么打）
3. 最后讨论目标和路径（要什么、怎么做）

如果直接跳到目标层，就是目标会，不是战略会。

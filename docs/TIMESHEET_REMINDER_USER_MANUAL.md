## 系统简介

工时提醒系统是一个智能化的工时管理辅助工具，帮助您：

✅ 及时填报工时，避免遗漏  
✅ 了解自己的工时异常情况  
✅ 提高工时填报质量  
✅ 避免审批超时  

## 主要功能

### 1. 📅 未填报工时提醒

**每天早上9点**，系统会检查您是否填报了昨天的工时。如果未填报，您会收到提醒。

**示例提醒**：
```
标题：工时填报提醒：2024-02-13
内容：您还未填报 2024年02月13日 的工时，请及时填报。
```

### 2. ⚠️ 异常工时预警

**每天下午2点**，系统会检测您的工时是否存在异常情况：

| 异常类型 | 说明 | 严重程度 |
|---------|------|---------|
| 单日超过12小时 | 单日总工时 > 12小时 | ⚠️ 警告 |
| 无效工时数据 | 工时 < 0 或 > 24 | ❌ 错误 |
| 周工时超标 | 一周总工时 > 60小时 | ⚠️ 警告 |
| 连续工作无休息 | 连续7天工作 | ⚠️ 警告 |
| 进度不匹配 | 工时与任务进度不符 | ℹ️ 提示 |

### 3. ⏰ 审批超时提醒

当您的工时提交超过**3天**仍未审批时，系统会提醒审批人及时处理。

### 4. 📊 工时统计Dashboard

提供可视化的工时统计信息：
- 待处理提醒数量
- 异常工时统计
- 历史提醒记录
- 紧急事项列表

## 提醒类型说明

### 🔔 未填报工时提醒

**触发条件**：昨天没有填报工时记录

**接收时间**：每天上午9:00

**处理方式**：
1. 进入工时填报页面
2. 选择日期，填写工时
3. 提交审批

**忽略提醒**：如果昨天确实无需填报工时（如请假、休息），可以忽略该提醒。

### ⚠️ 单日工时超标（>12小时）

**触发条件**：单日总工时超过12小时

**示例**：
```
异常类型：单日工时超标
日期：2024-02-13
实际工时：14.5小时
阈值：12小时
```

**建议处理**：
- 检查是否重复填报
- 确认是否有加班审批
- 如确实加班，可标记为"已解决"

### ❌ 无效工时数据

**触发条件**：工时 < 0 或 > 24

**严重程度**：错误

**建议处理**：
- 立即修正工时数据
- 检查填报流程是否有误

### ⚠️ 周工时超标（>60小时）

**触发条件**：一周（周一至周日）总工时超过60小时

**示例**：
```
异常类型：周工时超标
周期：2024-02-05 ~ 2024-02-11
实际工时：68小时
阈值：60小时
```

**建议处理**：
- 检查是否有连续加班情况
- 注意劳逸结合
- 向管理层反映工作量问题

### ⚠️ 连续工作无休息

**触发条件**：连续7天都有工时记录

**建议处理**：
- 检查是否确实连续工作
- 适当安排休息时间
- 注意身体健康

### ℹ️ 工时与进度不匹配

**触发条件**：
- 填报超过4小时但任务进度未更新
- 填报超过8小时但进度增加少于10%

**示例**：
```
异常类型：工时与进度不匹配
日期：2024-02-13
填报工时：8小时
进度变化：50% → 50%（无变化）
```

**建议处理**：
- 检查是否忘记更新进度
- 确认工时填报是否准确
- 如遇到困难导致进度缓慢，可以备注说明

## 操作指南

### 查看待处理提醒

1. 登录系统
2. 进入【工时管理】→【提醒中心】
3. 查看【待处理提醒】列表

**快捷筛选**：
- 按提醒类型筛选
- 按优先级筛选（紧急/高/普通/低）
- 按日期范围筛选

### 处理提醒

#### 方式1：处理问题

1. 点击提醒详情
2. 根据提醒内容，进入相关页面
3. 完成操作（如填报工时、修正数据）
4. 系统自动标记为"已解决"

#### 方式2：忽略提醒

如果提醒不适用于您的情况：

1. 点击【忽略】按钮
2. 填写忽略原因（可选）
3. 确认忽略

**示例**：
```
忽略原因：昨天请假，无需填报工时
```

### 查看异常记录

1. 进入【工时管理】→【异常记录】
2. 查看未解决的异常列表
3. 点击异常详情查看具体信息

**异常详情包括**：
- 异常类型
- 检测时间
- 相关工时记录
- 异常数据详情
- 严重程度

### 解决异常

1. 打开异常详情
2. 根据异常类型采取相应操作
3. 点击【标记为已解决】
4. 填写解决说明（可选）

**示例**：
```
解决说明：已修正重复填报的工时记录
```

### 查看历史记录

1. 进入【工时管理】→【提醒历史】
2. 选择时间范围
3. 查看历史提醒和处理情况

**可导出报表**：
- 个人工时提醒统计
- 异常工时统计
- 处理效率分析

### 查看Dashboard

1. 进入【工时管理】→【提醒Dashboard】
2. 查看可视化统计信息

**Dashboard包含**：
- 📊 提醒统计
  - 总提醒数
  - 待处理数
  - 已解决数
  - 已忽略数
  
- 🔍 异常统计
  - 总异常数
  - 未解决数
  - 按类型分布
  - 按严重程度分布

- 📋 最近提醒
  - 最近10条提醒
  - 紧急事项列表

## 常见问题

### Q1: 为什么我会收到重复的提醒？

**A**: 系统默认每日最多发送1次同类型提醒。如果收到重复提醒，可能是：
- 规则配置允许多次提醒
- 不同类型的提醒（未填报 vs 异常）
- 前一次提醒已被忽略

**解决方法**：检查规则配置的 `max_reminders_per_day` 设置。

### Q2: 我已经填报了工时，为什么还收到提醒？

**A**: 可能原因：
- 工时状态为"草稿"，未提交
- 系统检测时间早于您填报时间
- 缓存未及时更新

**解决方法**：
1. 确认工时状态为"已提交"或"已通过"
2. 刷新页面查看最新状态
3. 如确实已填报，可忽略该提醒

### Q3: 如何关闭某类提醒？

**A**: 普通用户无法自行关闭提醒。如需关闭，请联系系统管理员调整规则配置。

### Q4: 异常检测的阈值可以调整吗？

**A**: 阈值由系统管理员在规则配置中设置，普通用户无法修改。如有特殊需求，可向管理员申请调整。

### Q5: 提醒会保留多久？

**A**: 
- 待处理提醒：一直保留直到处理或忽略
- 历史记录：默认保留1年
- 异常记录：默认保留6个月

### Q6: 如何查看团队的整体提醒情况？

**A**: 
- 普通用户只能查看个人提醒
- 部门管理员可查看部门统计
- 系统管理员可查看全局统计

### Q7: 周末和节假日会收到提醒吗？

**A**: 
- 未填报提醒：默认不在周末/节假日发送
- 异常预警：会检测周末/节假日的工时异常
- 可由管理员配置规则

### Q8: 提醒能否自定义？

**A**: 
- 提醒内容：由管理员配置模板
- 提醒时间：由定时任务配置
- 通知渠道：可在个人设置中选择

### Q9: 如果对异常检测结果有异议怎么办？

**A**: 
1. 标记异常为"已解决"并填写说明
2. 联系直属领导确认
3. 如确有问题，联系管理员调整规则

### Q10: 如何提高工时填报质量？

**A**: 
1. 每天及时填报，避免遗忘
2. 填报时同步更新任务进度
3. 注意工时数据的合理性
4. 定期查看Dashboard了解自己的工时习惯

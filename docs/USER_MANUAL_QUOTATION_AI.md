## 功能概述

AI报价单自动生成器是一款智能化的售前报价工具，帮助销售和售前团队快速生成专业、准确的报价单。

### 核心功能

✅ **一键生成报价单** - 自动填充客户信息、计算价格、生成付款条款  
✅ **三档方案生成** - 智能生成基础版、标准版、高级版三套方案  
✅ **PDF自动导出** - 专业模板、公司Logo、规范格式  
✅ **版本管理** - 历史追踪、变更对比、一键回滚  
✅ **审批流程** - 多级审批、意见反馈、状态追踪  
✅ **邮件发送** - 一键发送PDF报价单给客户  

### 适用场景

- 软件开发项目报价
- 系统集成方案报价
- 定制化产品报价
- 技术服务报价
- 售前技术方案

## 快速开始

### 1. 生成第一个报价单

#### 步骤1: 准备报价项

确定您要报价的项目，包括：

## 功能详解

### 1. 报价单生成

#### 报价类型

**基础版（Basic）**
- 适用场景: 小型项目、标准化产品
- 特点: 简洁明了、快速交付
- 折扣: 通常无折扣
- 示例: 基础版ERP（¥80,000）

**标准版（Standard）**
- 适用场景: 中型项目、定制化需求
- 特点: 功能完整、性价比高
- 折扣: 5-10%
- 示例: 标准版ERP + 移动端（¥180,000）

**高级版（Premium）**
- 适用场景: 大型项目、复杂系统
- 特点: 全功能、VIP服务
- 折扣: 10-15%
- 示例: 高级版ERP + 定制开发 + 系统集成（¥350,000）

#### 价格计算

系统自动计算以下项目：

1. **小计（Subtotal）** = Σ(数量 × 单价)
2. **税费（Tax）** = 小计 × 税率
3. **折扣（Discount）** = 小计 × 折扣率
4. **总计（Total）** = 小计 + 税费 - 折扣

示例：
```
小计: ¥150,000
税费: ¥19,500 (13%)
折扣: -¥7,500 (5%)
─────────────────
总计: ¥162,000
```

#### 付款条款

**自动生成**（推荐）
系统根据报价类型自动生成合理的付款条款：

- 基础版: 一次性支付
- 标准版: 30% + 40% + 30%
- 高级版: 20% + 30% + 30% + 20%

**自定义**
您也可以自行编辑付款条款，例如：
```
总金额: ¥162,000
付款方式:
- 首付款: 40% (¥64,800) - 签订合同后5个工作日内
- 中期款: 40% (¥64,800) - 完成中期验收后5个工作日内
- 尾款: 20% (¥32,400) - 完成终期验收后5个工作日内
```

### 3. 版本管理

#### 为什么需要版本管理？

- 客户要求调整价格
- 需求变更导致报价项变化
- 审批后需要修改
- 保留历史记录

#### 版本追踪

每次修改报价单，系统会：

1. 自动递增版本号（V1 → V2 → V3）
2. 创建完整的版本快照
3. 记录变更摘要
4. 保存变更人和时间

示例版本历史：
```
V3 (2026-02-15 14:00) - 用户A
  变更: 调整折扣率; 有效期调整为60天

V2 (2026-02-15 12:00) - 用户A
  变更: 更新报价项

V1 (2026-02-15 10:00) - 用户A
  变更: 初始创建
```

#### 查看版本历史

1. 进入报价单详情页
2. 点击"版本历史"标签
3. 查看所有历史版本
4. 点击任意版本查看快照详情

#### 版本对比

选择两个版本进行对比，系统会高亮显示：
- 新增的报价项（绿色）
- 删除的报价项（红色）
- 修改的字段（黄色）

### 4. 审批流程

#### 审批状态

报价单有以下状态：

- **草稿（Draft）** - 初始创建
- **待审批（Pending Approval）** - 提交审批
- **已审批（Approved）** - 审批通过
- **已拒绝（Rejected）** - 审批拒绝
- **已发送（Sent）** - 发送给客户
- **已接受（Accepted）** - 客户接受
- **已拒绝（Rejected）** - 客户拒绝

#### 审批流程

**步骤1: 提交审批**
1. 报价单创建完成后
2. 点击"提交审批"
3. 状态变为"待审批"

**步骤2: 审批人处理**
1. 审批人收到通知
2. 查看报价单详情
3. 选择"批准"或"拒绝"
4. 填写审批意见

**步骤3: 结果处理**
- **批准**: 状态变为"已审批"，可以发送给客户
- **拒绝**: 状态变为"已拒绝"，需要修改后重新提交

#### 多级审批

支持配置多级审批：
- 第一级: 部门经理
- 第二级: 总监
- 第三级: 副总裁（大额报价单）

## 最佳实践

### 1. 报价单命名规范

建议使用统一的编号规则：
```
QT-YYYYMMDD-XXXX

QT: Quotation（报价单）
YYYYMMDD: 日期
XXXX: 当天第X个报价单（自动递增）

示例: QT-20260215-0001
```

### 2. 报价项分类

建议按照以下分类组织报价项：

- **软件开发** - 系统开发、模块开发
- **硬件设备** - 服务器、网络设备
- **服务** - 部署、培训、技术支持
- **许可证** - 软件许可、第三方组件
- **其他** - 差旅、文档等

### 3. 定价策略

**基础版**
- 成本价 × 1.5
- 无折扣
- 吸引预算有限的客户

**标准版**（重点）
- 成本价 × 2.0
- 5-10% 折扣
- 性价比最高，推荐方案

**高级版**
- 成本价 × 2.5
- 10-15% 折扣
- 高利润，展示专业能力

### 4. 有效期设置

根据项目复杂度设置合理的有效期：

- **简单项目**: 30天
- **中等项目**: 45-60天
- **复杂项目**: 90天

### 5. 付款节点

建议设置3-4个付款节点：

**3节点（推荐）**
```
首付: 30%  - 合同签订后
中期: 40%  - 中期验收后
尾款: 30%  - 最终验收后
```

**4节点（大型项目）**
```
首付: 20%  - 合同签订后
需求确认: 30%  - 需求文档签署后
中期: 30%  - 中期验收后
尾款: 20%  - 最终验收后
```

## 常见问题

### Q1: 如何修改已发送的报价单？

**A**: 已发送的报价单可以修改，但会创建新版本：

1. 点击"编辑"按钮
2. 修改需要调整的内容
3. 保存后版本号自动递增
4. 重新导出PDF并发送给客户
5. 说明变更原因

### Q2: 客户要求降价，如何处理？

**A**: 有两种方式：

**方式1: 调整折扣率**
1. 编辑报价单
2. 增加折扣率（如从5%调整到10%）
3. 保存并重新生成PDF

**方式2: 调整报价项**
1. 减少某些非必需功能
2. 或降低某些项目的单价
3. 保存并重新生成PDF

### Q3: 如何回滚到历史版本？

**A**: 系统不直接支持回滚，但可以：

1. 查看版本历史
2. 找到要恢复的版本
3. 复制该版本的数据
4. 创建新报价单
5. 粘贴数据并保存

### Q4: PDF中文显示乱码怎么办？

**A**: 需要配置中文字体：

1. 联系管理员
2. 安装中文字体文件
3. 在系统设置中配置字体路径
4. 重新生成PDF

### Q5: 一天可以生成多少个报价单？

**A**: 没有限制，但建议：

- 合理规划报价单编号
- 定期清理草稿状态的报价单
- 超过100个/天可能需要优化

### Q6: 如何导出所有报价单数据？

**A**: 使用数据导出功能：

1. 进入报价单列表页
2. 选择日期范围
3. 点击"导出Excel"
4. 下载包含所有报价单的Excel文件

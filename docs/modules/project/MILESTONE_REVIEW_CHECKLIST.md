# 短期里程碑审阅清单

> 依据 `docs/SHORT_TERM_BURNDOWN_PLAN.md` 中的 W0-W6 路线，梳理各里程碑的进入条件、验证项与输出物，便于 PMO 在周例会/专项评审中复用。

## 1. 时间线概览

| 里程碑 | 周次 | 聚焦主题 | 目标摘要 |
| --- | --- | --- | --- |
| M0 | W0 启动 | Kickoff | 对齐人力、SP 基线与依赖项 |
| M1 | W1 | 调度治理 | 健康度服务梳理、调度策略确认 |
| M2 | W2 | Scheduler 回归 | 24 个定时服务验证闭环 |
| M3 | W3 | 前端/DB 联调 | 生产/验收页 Demo + 缺料表迁移初稿 |
| M4 | W4 | UAT Checkpoint | 移动端适配样例、缺料/研发表数据导入 |
| M5 | W5 | 数据字典评审 | 研发项目 6 张表落地、字段说明确认 |
| M6 | W6 收尾 | 发布审批 | 回归测试、燃尽复盘、上线物料 |

## 2. 审阅项明细

### M1 – 调度治理
- **进入条件**：Kickoff 完成、人力与 SP 锁定；`ENABLE_SCHEDULER` 策略讨论会已排期。
- **检查清单**：
  1. 健康度指标补齐方案（含口径、字段映射、回滚策略）。
  2. 24 个定时任务现状盘点表（触发频率、负责人、监控方式）。
  3. 调度框架增强需求列表：热切换、失败重试、运行日志。
- **输出物**：会议纪要、任务拆解确认清单、更新后的燃尽图。

### M2 – Scheduler 回归
- **进入条件**：M1 输出物已归档，核心任务进入开发。
- **检查清单**：
  1. `test_scheduled_services.py` 全部通过并附回归日志。
  2. 预警服务通知链路测试记录（短信/企微/邮件）。
  3. 调度监控日志示例（含失败重试、任务状态）。
- **输出物**：回归报告、监控截图、问题/风险列表。

### M3 – 前端 & 数据库联合评审
- **进入条件**：生产/验收页面功能开发完成 80%+；缺料/研发表迁移脚本初稿可运行。
- **检查清单**：
  1. 生产管理四大视图 Demo（计划、执行、报工、异常）录屏。
  2. 验收管理页面联调记录（含 API 清单、异常项）。
  3. Alembic 迁移脚本审阅（缺料 2 表 + 研发项目 6 表）及数据样例。
- **输出物**：联调纪要、脚本评审意见、后续修订计划。

### M4 – UAT Checkpoint
- **进入条件**：W3 事项缺陷已关闭、Staging 环境具备联调数据。
- **检查清单**：
  1. 移动端适配页面 Demo（至少 2 个关键页面）。
  2. 缺料/研发表真实数据导入报告（含校验脚本结果）。
  3. QA/UAT 用例更新：新增移动端与数据一致性测试。
- **输出物**：UAT 会议纪要、缺陷列表、下一步上线准备事项。

### M5 – 数据字典评审
- **进入条件**：数据库迁移上线到测试环境，主流程数据跑通。
- **检查清单**：
  1. `docs/数据字典.md` 同步最新字段（含默认值、索引、权限说明）。
  2. 研发项目 6 张表与业务方确认字段含义及数据来源。
  3. 数据一致性校验脚本结果（覆盖新增表）。
- **输出物**：数据字典评审纪要、遗留字段列表、风险评估。

### M6 – 发布审批
- **进入条件**：剩余 SP≤2，关键缺陷关闭，回归测试 100% 通过。
- **检查清单**：
  1. 回归测试报告（后台定时、前端页面、数据迁移）。
  2. 燃尽复盘：实际 vs 计划 SP，对偏差给出原因与改进点。
  3. 上线变更单（部署步骤、回滚方案、值班表）。
- **输出物**：发布审批材料、复盘文档、知识库更新（FAQ/操作指南）。

## 3. 审阅节奏建议
1. **周例会**：按燃尽表同步剩余 SP、阻塞项，与 owner 更新 checklist。
2. **专项里程碑会**：在里程碑前 1-2 天组织评审，确保输出物提前准备。
3. **记录归档**：所有纪要、报告统一沉淀到 `docs/` 或项目知识库，方便审计。

## 4. 风险与应对
- **跨团队依赖**：前端/数据/后端需确认接口冻结窗口，避免互相阻塞 → 在 M1 制定接口变更流程。
- **人力波动**：若 owner 请假/转岗，提前在周会中安排 backup，必要时调整 SP 基线。
- **新增 P0 需求**：触发即召开临时评审，决定是否拉长燃尽周期或增援人手。

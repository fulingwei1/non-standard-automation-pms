# ECN 物料成本影响跟踪使用指南

## 功能概述

设计变更（ECN）会导致物料变更和成本变化。本功能帮助你：
- 📊 分析 ECN 对物料的影响
- 💰 跟踪 ECN 导致的成本变化
- 👥 通知相关人员并跟踪执行进度
- 📈 评估对项目交付的影响

---

## 核心功能

### 1. ECN 物料影响分析

**入口**: ECN 管理 → 选择 ECN → 物料影响分析

**操作步骤**:
1. 打开 ECN 管理页面
2. 选择需要分析的 ECN
3. 点击"物料影响分析"按钮
4. 系统自动分析受影响的物料

**分析结果**:
- 受影响物料清单
- 物料当前状态（未采购/已下单/在途/已入库）
- 潜在损失金额
- 影响的采购项目
- 对项目交付的影响评估

**API**: `POST /api/v1/ecns/{ecn_id}/material-impact-analysis`

---

### 2. ECN 成本影响分析

**入口**: ECN 管理 → 选择 ECN → 成本影响分析

**操作步骤**:
1. 打开 ECN 管理页面
2. 选择需要分析的 ECN
3. 点击"成本影响分析"按钮
4. 查看成本分析结果

**分析结果**:
- 直接成本（物料报废/返工/新购/索赔）
- 间接成本（延期/管理）
- 总成本影响
- 按成本类型分类统计
- 成本影响最大的物料 TOP10

**成本类型说明**:
| 类型 | 说明 | 示例 |
|------|------|------|
| 报废 | 已购物料无法使用 | 价值 5 万的物料因 ECN 报废 |
| 返工 | 已装配部分需要返工 | 返工人工费 2 万 |
| 新购 | ECN 新增物料 | 新增物料采购 3 万 |
| 索赔 | 向供应商索赔 | 供应商赔偿 1 万 |
| 延期 | 项目延期成本 | 延期罚款 5 万 |
| 管理 | ECN 执行管理成本 | 管理人员工时 1 万 |

**API**: `POST /api/v1/ecns/{ecn_id}/cost-impact-analysis`

---

### 3. 记录 ECN 成本

**入口**: ECN 管理 → 选择 ECN → 成本记录 → 新增

**操作步骤**:
1. 打开 ECN 详情页面
2. 切换到"成本记录"标签
3. 点击"新增成本记录"
4. 填写成本信息：
   - 成本类型（报废/返工/新购/延期）
   - 金额
   - 原因说明
   - 关联物料（可选）
   - 上传凭证（发票/单据）
5. 提交审批

**审批流程**:
- 金额 < 1 万：采购经理审批
- 金额 ≥ 1 万：财务总监审批
- 金额 ≥ 10 万：总经理审批

**API**: `POST /api/v1/ecns/{ecn_id}/cost-records`

---

### 4. ECN 执行进度跟踪

**入口**: ECN 管理 → 选择 ECN → 执行进度

**执行阶段**:
1. 通知供应商（10%）
2. 采购变更（20%）
3. 物料处理（40%）
4. 验证确认（20%）
5. 关闭（10%）

**跟踪内容**:
- 每个阶段的完成状态
- 每个受影响物料的处理状态
- 预计完成日期
- 阻塞问题列表

**API**: `GET /api/v1/ecns/{ecn_id}/execution-progress`

---

### 5. 相关人员通知

**自动通知场景**:
- ECN 发布时
- 物料影响分析完成
- 成本记录提交审批
- 执行进度关键节点
- 物料处理状态变化

**相关人员**:
- 项目经理
- 采购员
- 供应商联系人
- 设计人员
- 审批人

**订阅管理**:
- 进入 ECN 详情页
- 点击"通知订阅"
- 选择要接收的通知类型
- 保存订阅设置

**API**: `POST /api/v1/ecns/{ecn_id}/notify-stakeholders`

## 使用场景

### 场景 1: ECN 发布后的物料影响评估

**背景**: 客户提出设计变更，需要评估影响

**步骤**:
1. 创建 ECN 并提交审批
2. ECN 批准后，点击"物料影响分析"
3. 查看受影响物料清单和状态
4. 评估对项目交付的影响
5. 通知相关人员（项目经理/采购员/供应商）

**输出**:
- 受影响物料清单
- 潜在损失金额
- 预计延期天数
- 相关方通知记录

---

### 场景 2: ECN 成本跟踪

**背景**: ECN 执行过程中需要跟踪实际成本

**步骤**:
1. ECN 批准后，点击"成本影响分析"预估总成本
2. 执行过程中，逐项录入实际成本
3. 上传凭证（发票/单据）
4. 提交审批
5. 查看预算 vs 实际对比

**输出**:
- 成本影响分析报告
- 实际成本记录
- 预算执行率
- 成本趋势图

---

### 场景 3: ECN 执行进度管理

**背景**: 需要跟踪 ECN 执行进度，确保按时完成

**步骤**:
1. ECN 批准后，点击"开始执行"
2. 按计划执行各阶段任务
3. 每完成一个阶段，更新进度
4. 记录物料处理决策（继续使用/返工/报废/退货）
5. 执行完成后，点击"验证"和"关闭"

**输出**:
- 执行进度报告
- 物料处理记录
- 实际损失统计
- ECN 执行日志

## 常见问题

### Q1: 如何识别受影响的采购订单？

**A**: 系统会自动分析：
- 已下单但未到货的订单
- 正在生产中的物料
- 已入库但未使用的物料

在"物料影响分析"结果中会列出所有受影响的采购订单。

---

### Q2: 成本记录必须上传凭证吗？

**A**: 建议上传，但不是必须。以下情况可以免凭证：
- 预估成本（标记为"预估"）
- 内部成本（人工/管理）
- 金额 < 1000 元

但正式成本记录必须上传凭证才能通过审批。

---

### Q3: 如何通知供应商？

**A**: 系统支持多种通知方式：
- 站内通知（系统内）
- 邮件通知（需配置邮箱）
- 短信通知（需配置短信服务）
- 微信通知（需配置企业微信）

在"相关人员通知"中选择通知渠道。

---

### Q4: ECN 执行进度如何更新？

**A**: 有两种方式：
1. **手动更新**: 在执行进度页面，点击阶段旁的"完成"按钮
2. **自动更新**: 当物料处理决策提交后，自动更新"物料处理"阶段进度

---

### Q5: 如何查看项目所有 ECN 的成本汇总？

**A**: 进入项目详情页，切换到"成本"标签，查看"ECN 成本汇总"：
- 所有 ECN 的成本汇总
- 按 ECN 分类统计
- 按成本类型统计
- ECN 成本占项目总成本比例

**API**: `GET /api/v1/projects/{id}/costs/ecn-summary`

## 最佳实践

### 1. 尽早分析影响

ECN 批准后**立即**进行物料影响分析，不要等到执行阶段才发现问题。

### 2. 准确记录成本

每笔成本发生后**及时录入**，避免月底集中补录导致数据不准确。

### 3. 保持信息同步

确保相关人员都**订阅了通知**，关键决策要及时通知到所有人。

### 4. 定期跟踪进度

建议**每周检查**一次 ECN 执行进度，识别阻塞问题并及时解决。

### 5. 善用成本预警

设置成本预警阈值，当实际成本超过预算**10%**时自动预警。

# 非标自动化测试设备项目管理系统 - 设计文档

## 一、系统概述

### 1.1 适用范围
本系统专为非标自动化测试设备和线体制造企业设计，适用于：
- ICT/FCT/EOL测试设备
- 烧录设备、老化设备
- 视觉检测设备
- 自动化组装线体
- 设备改造、软件开发项目

### 1.2 设计目标
1. **全流程管理**：覆盖从签单到售后的完整项目生命周期
2. **多维度追踪**：项目、人员、物料、客户、财务多维度数据联动
3. **异常预警**：自动识别逾期风险，及时预警
4. **数据分析**：提供统计报表，支持决策

## 三、核心表单详细设计

### 3.1 项目主表（核心）

#### 字段分组设计：

| 分组 | 字段 | 说明 |
|------|------|------|
| **基础信息** | 序号、项目号、合同号、优先级、项目状态、预警标识、项目名称 | 项目唯一标识与状态 |
| **客户信息** | 客户名称、联系人、联系电话、收货地址 | 交付与沟通信息 |
| **项目分类** | 项目种类、设备类型、是否标准机 | 便于统计分析 |
| **时间节点** | 签单日期、合同交期、协商交期、计划/实际出BOM、计划/实际到料、计划出货 | 关键里程碑 |
| **人员分工** | 业务、PMC、项目经理、机械、电气、上位机/视觉、调试、售后 | 责任明确 |
| **进度状态** | 机台数、已出货、待交、当前阶段、阶段完成率、整体进度 | 实时进度 |
| **财务信息** | 合同金额、已收款、待收款、付款状态 | 收款跟踪 |
| **风险备注** | 风险等级、逾期原因、当前状况 | 问题追踪 |

#### 下拉选项定义：

**优先级**
- A-特急：客户投诉/紧急项目，当天响应
- B-紧急：交期临近，3天内响应
- C-正常：按计划推进
- D-待定：暂停或待启动项目

**项目状态**
- 进行中、已完成、暂停、取消、待启动

**项目种类**
- 单机类：独立测试设备
- 线体类：自动化产线
- 改造类：现有设备改造升级
- 软件类：纯软件开发
- 工装类：治具/夹具
- 贸易类：外购设备配套
- 物料类：零配件/备件

**设备类型**
- ICT测试设备、FCT测试设备、EOL测试设备
- 烧录设备、老化设备、视觉检测、组装线体、其他

**当前阶段**
- 需求确认 → 方案设计 → 出BOM → 采购下单 → 物料到货 → 装配 → 调试 → 验收 → 出货 → 售后

### 3.2 生产进度跟踪

专注于生产过程的关键节点管理：

| 阶段 | 计划日期 | 实际日期 | 延期天数 | 完成率 |
|------|----------|----------|----------|--------|
| 出BOM | ✓ | ✓ | 自动计算 | - |
| 到料 | ✓ | ✓ | 自动计算 | 到料率% |
| 装配 | ✓ | ✓ | 自动计算 | - |
| 调试 | ✓ | ✓ | 自动计算 | - |

**关键公式**：
```excel
延期天数 = IF(AND(实际日期<>"",计划日期<>""), 实际日期-计划日期, "")
到料率 = 已到货物料数 / BOM物料总数 × 100%
```

### 3.3 人员工作量统计

用于评估团队负荷，合理分配资源：

| 指标 | 说明 |
|------|------|
| 在进行项目数 | 当前负责的未完成项目 |
| 已完成项目数 | 历史完成量 |
| 本月出货目标/已出货 | 月度KPI |
| 完成率% | 出货/目标 |
| 平均项目周期 | 效率指标 |
| 逾期项目数 | 质量指标 |
| 负荷评估 | 空闲/正常/适中/繁忙/超负荷 |

### 3.4 物料跟踪表

采购到货全流程管理：

**物料类别**：机加件、钣金件、标准件、电气件、气动元件、外协件、贸易件

**到货状态**：未下单 → 已下单 → 部分到货 → 已到货 → 已入库 → 有问题

**关键字段**：
- 供应商信息
- 预计到货 vs 实际到货
- 延期天数自动计算
- 问题物料标注

### 3.5 客户交付看板

按客户维度汇总，支持客户关系管理：

| 维度 | 指标 |
|------|------|
| 项目 | 进行中数、已完成数 |
| 财务 | 总合同金额、已收款、待收款 |
| 交付 | 本月待交付、逾期项目数 |
| 服务 | 客户满意度、最近/下次交付日期 |

### 3.6 异常项目管理

专项跟踪问题项目：

**异常类型**：
- 出货逾期、物料延期、设计变更
- 客户暂停、质量问题、资金问题
- 产能不足、其他

**跟踪要素**：
- 异常等级（紧急/重要/一般）
- 责任部门/责任人
- 解决措施
- 预计/实际解决日期
- 跟进记录

## 四、数据规范

### 4.1 编码规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 项目号 | PJyymmddxxx | PJ250708001 |
| 合同号 | HTyymm-xxx | HT2507-001 |
| 机台号 | PNxxx | PN001 |

### 4.2 日期格式
统一使用 **YYYY-MM-DD** 格式

### 4.3 金额格式
- 单位：元（大金额可用万元，需在表头标注）
- 保留2位小数

## 五、使用建议

### 5.1 日常维护流程

```
每日更新：
1. 更新项目当前阶段和进度
2. 更新物料到货状态
3. 检查并处理预警项目

每周汇总：
1. 更新人员工作量统计
2. 审核异常项目处理情况
3. 更新客户交付看板

每月分析：
1. 完成月度统计仪表盘
2. 分析KPI达成情况
3. 总结问题和改进措施
```

### 5.2 条件格式建议

在Excel中设置以下条件格式：
- **红色底色**：A-特急项目、高风险项目
- **黄色底色**：B-紧急项目、中风险项目
- **绿色底色**：已完成项目
- **蓝色底色**：进行中项目

### 5.3 筛选与排序

建议的默认视图：
1. 按优先级排序（A→D）
2. 按合同交期排序（近→远）
3. 筛选"进行中"状态项目

## 修复内容总结

### 1. test_progress_integration_service.py

#### 问题1: AlertRecord id 字段问题（SQLite BigInteger autoincrement）
- **问题**: SQLite 在处理 `BigInteger` 的 `autoincrement` 时有问题。SQLite 只支持 `INTEGER PRIMARY KEY AUTOINCREMENT`，不支持 `BIGINT AUTOINCREMENT`。当之前的测试失败时，session 可能处于 rollback 状态。
- **修复**: 
  - 在所有创建 `AlertRecord` 的测试方法开始时添加 `db_session.rollback()` 清理状态
  - 在 `commit()` 之前添加 `flush()` 确保 `id` 生成
- **影响**: `test_shortage_alert` fixture 和多个测试方法

#### 问题2: 方法参数类型和返回值类型错误
- **问题**: 
  - `handle_acceptance_failed` 和 `handle_acceptance_passed` 接受 `AcceptanceOrder` 对象，但测试传入的是 `int`
  - `handle_ecn_approved` 返回 `Dict`，但测试期望 `list`
- **修复**: 
  - 创建 `AcceptanceOrder` 对象而不是传入整数
  - 调整断言，期望 `Dict` 而不是 `list`
- **影响**: `test_handle_acceptance_failed_not_found`, `test_handle_acceptance_passed_not_found`, `test_handle_ecn_approved_no_project`

#### 问题3: check_milestone_completion_requirements 参数类型
- **问题**: 方法接受 `ProjectMilestone` 对象，但测试传入的是 `int`
- **修复**: 创建 `ProjectMilestone` 对象或调整测试逻辑
- **影响**: `test_check_milestone_completion_requirements_not_found`

### 2. test_status_transition_service.py

#### 问题1: patch 路径错误
- **问题**: 测试中 patch 的是 `app.services.status_transition_service.check_s3_to_s4_transition`，但实际代码中是从 `app.services.stage_transition_checks` 导入的
- **修复**: 将所有 patch 路径改为 `app.services.stage_transition_checks.*`
- **影响**: 所有 `test_check_auto_stage_transition_*` 测试方法

#### 问题2: execute_stage_transition 返回值格式
- **问题**: `execute_stage_transition` 返回的字典格式与测试期望不匹配
- **修复**: 调整 mock 返回值格式，使用 `can_advance`, `auto_advanced`, `target_stage` 等键
- **影响**: `test_check_auto_stage_transition_with_auto_advance`

#### 问题3: _log_status_change 需要 commit
- **问题**: `_log_status_change` 方法只是 `add` 了日志，但没有 `commit`，所以查询时找不到
- **修复**: 在测试中调用 `commit()` 以保存日志
- **影响**: `test_log_status_change`, `test_log_status_change_minimal`

### 3. test_acceptance_completion_complete.py

#### 问题: Machine project_id 不能为 NULL
- **问题**: `Machine` 模型的 `project_id` 字段不能为 NULL，但测试中在创建 `Machine` 时 `project.id` 可能还未生成
- **修复**: 使用正则表达式批量修复，在所有创建 `Project` 后、创建 `Machine` 前添加 `flush()`，确保 `project.id` 可用；在创建 `Machine` 后、创建 `AcceptanceOrder` 前也添加 `flush()`，确保 `machine.id` 可用
- **影响**: 所有测试方法（16个地方需要添加 flush）

## 三、关键问题分析

### 3.1 Python版本兼容性问题

**问题**: `TypeError: unsupported operand type(s) for |: 'type' and 'type'`

**位置**: `app/services/evm_service.py:32`

**原因**: 使用了Python 3.10+的类型联合语法 `int | float`，但测试环境为Python 3.9

**影响**: 9个测试失败

**建议修复**:
```python
# 错误写法 (Python 3.10+)
from typing import Union
def calculate(value: int | float) -> float:
    pass

# 正确写法 (Python 3.9兼容)
from typing import Union
def calculate(value: Union[int, float]) -> float:
    pass
```

### 3.2 数据库完整性约束问题

**问题**: `IntegrityError: NOT NULL constraint failed: machines.project_id`

**位置**: 阶段推进服务测试

**原因**: 创建Machine对象时未正确设置project_id外键

**影响**: 4个测试失败/错误

**建议修复**:
```python
# 工厂函数需要确保project_id不为None
machine = MachineFactory(project_id=project.id)
```

### 3.3 模型字段不匹配问题

**问题**: `TypeError: 'from_status' is an invalid keyword argument for ProjectStatusLog`

**位置**: `test_project_full_lifecycle.py`

**原因**: ProjectStatusLog模型字段与测试代码不一致

**影响**: 2个测试失败

**建议**: 检查模型定义，确认字段名为 `old_status` 还是 `from_status`

### 3.4 缺失的API端点

**问题**: 多个测试因API端点未实现而失败

**缺失端点**:
- `/api/v1/projects/{project_id}/documents/*` (文档管理)
- `/api/v1/projects/{project_id}/members/*` (团队管理)
- `/api/v1/projects/{project_id}/milestones/report` (里程碑报告)

**影响**: 23个测试失败

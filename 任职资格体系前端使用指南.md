# 任职资格体系前端使用指南

## 一、访问路径

### 主页面
- **路径**: `/qualifications`
- **菜单**: HR经理工作台 → 组织管理 → 任职资格管理

## 二、功能模块

### 1. 等级管理

#### 功能说明
- 查看所有任职资格等级列表
- 创建、编辑、删除等级
- 按角色类型和状态筛选

#### 操作步骤

**查看等级列表**
1. 进入任职资格管理页面
2. 点击"等级管理"标签页
3. 使用筛选器筛选等级

**创建新等级**
1. 点击"新建等级"按钮
2. 填写等级信息：
   - 等级编码（如：ASSISTANT, JUNIOR, MIDDLE, SENIOR, EXPERT）
   - 等级名称（如：助理级、初级、中级、高级、专家级）
   - 排序顺序
   - 适用角色类型（可选）
   - 等级描述
3. 点击保存

**编辑等级**
1. 在等级列表中点击"编辑"图标
2. 修改等级信息
3. 点击保存

**删除等级**
1. 在等级列表中点击"删除"图标
2. 确认删除操作

> ⚠️ 注意：如果等级下存在关联的能力模型或员工认证，将无法删除

### 2. 能力模型管理

#### 功能说明
- 查看所有岗位能力模型
- 创建、编辑能力模型
- 按岗位类型和等级筛选

#### 操作步骤

**查看能力模型列表**
1. 点击"能力模型"标签页
2. 使用筛选器筛选模型

**创建新能力模型**
1. 点击"新建能力模型"按钮
2. 填写模型信息：
   - 岗位类型（ENGINEER/SALES/CUSTOMER_SERVICE/WORKER）
   - 岗位子类型（仅工程师需要，如：ME/EE/SW/TE）
   - 等级
   - 能力维度要求（JSON格式）
3. 点击保存

**查看能力模型详情**
1. 在模型列表中点击"查看"图标
2. 查看详细的能力维度要求

### 3. 员工认证管理

#### 功能说明
- 查看所有员工任职资格
- 认证员工任职资格
- 员工晋升评估
- 按岗位类型和状态筛选

#### 操作步骤

**查看员工任职资格列表**
1. 点击"员工认证"标签页
2. 使用筛选器筛选

**认证员工任职资格**
1. 点击"认证员工"按钮
2. 选择员工
3. 填写认证信息：
   - 岗位类型
   - 等级
   - 各维度评估详情
   - 认证日期
   - 有效期
4. 点击提交

**员工晋升评估**
1. 在员工列表中点击"晋升"图标
2. 选择目标等级
3. 填写评估详情
4. 系统自动检查晋升资格
5. 评估通过后自动更新等级

**查看员工详情**
1. 在员工列表中点击"查看"图标
2. 查看员工的任职资格详情和评估历史

## 三、界面说明

### 统计卡片
页面顶部显示4个统计卡片：
- **等级总数**: 系统中定义的任职资格等级数量
- **能力模型**: 岗位能力模型总数
- **已认证员工**: 获得任职资格的员工数量
- **待认证**: 待处理的认证申请数量

### 标签页
- **等级管理**: 管理任职资格等级
- **能力模型**: 管理岗位能力模型
- **员工认证**: 管理员工任职资格

### 筛选器
每个标签页都提供筛选功能：
- 等级管理：按角色类型、启用状态筛选
- 能力模型：按岗位类型筛选
- 员工认证：按岗位类型、认证状态筛选

### 操作按钮
- **查看**（眼睛图标）：查看详细信息
- **编辑**（铅笔图标）：编辑记录
- **删除**（垃圾桶图标）：删除记录（仅等级管理）
- **晋升**（上升箭头图标）：员工晋升评估（仅员工认证）

## 四、权限说明

### 可访问角色
- **HR经理**: 完整权限，可进行所有操作
- **管理员**: 完整权限
- **其他角色**: 仅查看权限（待实现）

### 操作权限
- **等级管理**: 仅HR可创建、编辑、删除
- **能力模型管理**: 仅HR可创建、编辑
- **员工认证**: 仅HR可认证和晋升
- **查看权限**: 所有用户可查看

```json
    "weight": 40,
    "items": [
```


## 六、常见问题

### Q1: 如何批量导入能力模型？

A: 目前需要逐个创建，批量导入功能待开发。

### Q2: 如何查看员工的评估历史？

A: 在员工认证列表中点击"查看"，可以查看该员工的所有评估记录。

### Q3: 能力模型可以复制吗？

A: 目前不支持复制功能，需要手动创建。建议先查看相似模型，然后创建新模型。

### Q4: 如何修改绩效融合权重？

A: 权重配置在后端服务中，需要修改 `app/services/performance_integration_service.py` 中的配置。

## 八、技术说明

### 使用的技术栈
- React 19
- React Router 7
- Tailwind CSS
- Radix UI 组件
- Framer Motion 动画
- Axios HTTP 客户端

### API 端点
所有 API 调用通过 `frontend/src/services/api.js` 中的 `qualificationApi` 对象进行。

### 页面组件
- 主页面: `frontend/src/pages/QualificationManagement.jsx`
- API 服务: `frontend/src/services/api.js` (qualificationApi)
- 路由配置: `frontend/src/App.jsx`
- 菜单配置: `frontend/src/components/layout/Sidebar.jsx`

# 非标自动化测试设备项目管理系统 - 设计文档

## 一、系统概述

### 1.1 适用范围
本系统专为非标自动化测试设备和线体制造企业设计，适用于：
- ICT/FCT/EOL测试设备
- 烧录设备、老化设备
- 视觉检测设备
- 自动化组装线体
- 设备改造、软件开发项目

### 1.2 设计目标
1. **全流程管理**：覆盖从签单到售后的完整项目生命周期
2. **多维度追踪**：项目、人员、物料、客户、财务多维度数据联动
3. **异常预警**：自动识别逾期风险，及时预警
4. **数据分析**：提供统计报表，支持决策

## 三、核心表单详细设计

### 3.1 项目主表（核心）

#### 字段分组设计：

| 分组 | 字段 | 说明 |
|------|------|------|
| **基础信息** | 序号、项目号、合同号、优先级、项目状态、预警标识、项目名称 | 项目唯一标识与状态 |
| **客户信息** | 客户名称、联系人、联系电话、收货地址 | 交付与沟通信息 |
| **项目分类** | 项目种类、设备类型、是否标准机 | 便于统计分析 |
| **时间节点** | 签单日期、合同交期、协商交期、计划/实际出BOM、计划/实际到料、计划出货 | 关键里程碑 |
| **人员分工** | 业务、PMC、项目经理、机械、电气、上位机/视觉、调试、售后 | 责任明确 |
| **进度状态** | 机台数、已出货、待交、当前阶段、阶段完成率、整体进度 | 实时进度 |
| **财务信息** | 合同金额、已收款、待收款、付款状态 | 收款跟踪 |
| **风险备注** | 风险等级、逾期原因、当前状况 | 问题追踪 |

#### 下拉选项定义：

**优先级**
- A-特急：客户投诉/紧急项目，当天响应
- B-紧急：交期临近，3天内响应
- C-正常：按计划推进
- D-待定：暂停或待启动项目

**项目状态**
- 进行中、已完成、暂停、取消、待启动

**项目种类**
- 单机类：独立测试设备
- 线体类：自动化产线
- 改造类：现有设备改造升级
- 软件类：纯软件开发
- 工装类：治具/夹具
- 贸易类：外购设备配套
- 物料类：零配件/备件

**设备类型**
- ICT测试设备、FCT测试设备、EOL测试设备
- 烧录设备、老化设备、视觉检测、组装线体、其他

**当前阶段**
- 需求确认 → 方案设计 → 出BOM → 采购下单 → 物料到货 → 装配 → 调试 → 验收 → 出货 → 售后

### 3.2 生产进度跟踪

专注于生产过程的关键节点管理：

| 阶段 | 计划日期 | 实际日期 | 延期天数 | 完成率 |
|------|----------|----------|----------|--------|
| 出BOM | ✓ | ✓ | 自动计算 | - |
| 到料 | ✓ | ✓ | 自动计算 | 到料率% |
| 装配 | ✓ | ✓ | 自动计算 | - |
| 调试 | ✓ | ✓ | 自动计算 | - |

**关键公式**：
```excel
延期天数 = IF(AND(实际日期<>"",计划日期<>""), 实际日期-计划日期, "")
到料率 = 已到货物料数 / BOM物料总数 × 100%
```

### 3.3 人员工作量统计

用于评估团队负荷，合理分配资源：

| 指标 | 说明 |
|------|------|
| 在进行项目数 | 当前负责的未完成项目 |
| 已完成项目数 | 历史完成量 |
| 本月出货目标/已出货 | 月度KPI |
| 完成率% | 出货/目标 |
| 平均项目周期 | 效率指标 |
| 逾期项目数 | 质量指标 |
| 负荷评估 | 空闲/正常/适中/繁忙/超负荷 |

### 3.4 物料跟踪表

采购到货全流程管理：

**物料类别**：机加件、钣金件、标准件、电气件、气动元件、外协件、贸易件

**到货状态**：未下单 → 已下单 → 部分到货 → 已到货 → 已入库 → 有问题

**关键字段**：
- 供应商信息
- 预计到货 vs 实际到货
- 延期天数自动计算
- 问题物料标注

### 3.5 客户交付看板

按客户维度汇总，支持客户关系管理：

| 维度 | 指标 |
|------|------|
| 项目 | 进行中数、已完成数 |
| 财务 | 总合同金额、已收款、待收款 |
| 交付 | 本月待交付、逾期项目数 |
| 服务 | 客户满意度、最近/下次交付日期 |

### 3.6 异常项目管理

专项跟踪问题项目：

**异常类型**：
- 出货逾期、物料延期、设计变更
- 客户暂停、质量问题、资金问题
- 产能不足、其他

**跟踪要素**：
- 异常等级（紧急/重要/一般）
- 责任部门/责任人
- 解决措施
- 预计/实际解决日期
- 跟进记录

### 3.7 里程碑与验收（交付物清单）

为避免“验收口径不清、资料不全、回款被动”，里程碑必须与交付物清单、验收标准绑定。

#### 3.7.1 里程碑分级

**L1 管理里程碑（管理层）**  
合同签订 → 方案评审 → 设计冻结 → 出厂 → 验收 → 结项

**L2 交付里程碑（项目经理）**  
需求确认 → 出BOM → 关键物料到齐 → 装配完成 → 调试完成 → FAT → 发货 → SAT → 终验收

**L3 任务里程碑（执行层）**  
机械/电气/软件具体任务与工时（见任务管理/WBS）。

#### 3.7.2 里程碑字段定义（建议）

| 字段 | 说明 |
|------|------|
| 里程碑名称 | 与阶段绑定（S1-S9） |
| 计划/实际日期 | 用于延期计算 |
| 负责人 | PM或阶段负责人 |
| 验收标准 | 对应验收模板/指标 |
| 交付物清单 | 必交付物集合 |
| 审批人/签字 | 内部/客户 |
| 回款关联 | 对应收款节点 |

#### 3.7.3 交付物清单（按阶段）

| 阶段 | 必交付物示例 |
|------|-------------|
| 需求确认 | 需求采集表、验收标准清单、接口清单 |
| 方案评审 | 方案评审纪要、风险清单、关键参数确认 |
| 设计冻结 | BOM Vx、图纸/电气原理图、软件功能清单 |
| 采购/装配 | 关键物料到齐清单、来料检验记录 |
| 调试/出厂 | 调试报告、FAT测试报告、问题清单闭环 |
| 发货/交付 | 发货单/装箱单、移交清单、培训签到 |
| SAT/终验收 | SAT报告、终验收报告、客户签字 |
| 结项 | 结项报告、质保计划/售后记录 |

#### 3.7.4 验收标准（建议模板）

**FAT（出厂验收）**
- 功能项/节拍/精度/安全互锁必须达标
- 关键异常与偏差必须闭环
- 输出：FAT报告 + 整改记录

**SAT（现场验收）**
- 现场环境适配、接口联调、节拍稳定性
- 客户现场操作与培训完成
- 输出：SAT报告 + 培训签到

**终验收**
- 合同范围与交付物齐全
- 争议项全部关闭或明确书面豁免
- 输出：终验收报告 + 签字

#### 3.7.5 规则（系统硬约束）

1. 里程碑完成必须上传全部交付物并签字确认。  
2. 交付物版本受控，变更需关联ECN并记录版本号。  
3. 与回款绑定的里程碑未完成，禁止发起开票/收款申请。  
4. 验收不通过必须生成问题清单并闭环后可重新申请验收。  

#### 3.7.6 验收表单字段（FAT/SAT/终验收）

| 字段 | 说明 |
|------|------|
| 验收类型 | FAT / SAT / 终验收 |
| 项目/机台 | 项目号 + 机台号 |
| 计划/实际验收日期 | 计划、实际 |
| 验收项清单 | 功能/节拍/精度/安全/接口 |
| 结果 | 通过/不通过 |
| 不通过原因 | 自动生成问题清单 |
| 客户签字 | 签字人/签字时间 |
| 备注 | 现场说明 |

#### 3.7.7 问题清单字段（验收不通过闭环）

| 字段 | 说明 |
|------|------|
| 问题编号 | 自动编号 |
| 问题类型 | 性能/安全/资料/外观 |
| 严重度 | 严重/一般 |
| 责任部门/人 | 责任归属 |
| 计划关闭日期 | SLA |
| 状态 | 未解决/已解决/已验证 |
| 关闭验证 | 复验记录 |

#### 3.7.8 交付物模板库字段

| 字段 | 说明 |
|------|------|
| 模板名称 | FAT报告/验收单/培训签到 |
| 适用阶段 | S1-S9 |
| 是否必交付 | 是/否 |
| 输出格式 | PDF/Word/Excel |
| 责任人 | 交付负责人 |
| 版本 | V1/V2 |

## 五、使用建议

### 5.1 日常维护流程

```
每日更新：
1. 更新项目当前阶段和进度
2. 更新物料到货状态
3. 检查并处理预警项目

每周汇总：
1. 更新人员工作量统计
2. 审核异常项目处理情况
3. 更新客户交付看板

每月分析：
1. 完成月度统计仪表盘
2. 分析KPI达成情况
3. 总结问题和改进措施
```

### 5.2 条件格式建议

在Excel中设置以下条件格式：
- **红色底色**：A-特急项目、高风险项目
- **黄色底色**：B-紧急项目、中风险项目
- **绿色底色**：已完成项目
- **蓝色底色**：进行中项目

### 5.3 筛选与排序

建议的默认视图：
1. 按优先级排序（A→D）
2. 按合同交期排序（近→远）
3. 筛选"进行中"状态项目

## 六、系统扩展建议

### 6.1 可增加的功能表单

1. **设备BOM模板库**：标准设备的BOM模板
2. **供应商管理**：供应商评价与历史记录
3. **设备售后记录**：设备售后服务跟踪
4. **项目利润分析**：成本核算与利润统计
5. **质量问题库**：常见问题与解决方案

### 6.2 系统化升级路径

当前Excel版本 → 未来可升级为：
1. **在线协作**：使用飞书/钉钉多维表格
2. **低代码平台**：宜搭、简道云等
3. **专业ERP**：金蝶、用友等
4. **自研系统**：基于n8n + 数据库的自动化系统

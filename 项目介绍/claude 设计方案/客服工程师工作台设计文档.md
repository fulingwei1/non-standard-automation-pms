# 客服工程师工作台设计文档

## 一、设计概述

### 1.1 目标用户
**客服工程师** (`customer_service_engineer`)

### 1.2 岗位职责
根据角色配置，客服工程师的主要职责包括：
- **客户技术支持**：为客户提供技术咨询和问题解答
- **问题处理**：处理客户反馈的问题，跟踪问题解决进度
- **问题工单管理**：**核心功能** - 创建问题工单，转派给相应的人员（项目经理、技术工程师等）
- **现场服务**：执行现场安装、调试、培训等服务任务
- **客户沟通**：维护客户关系，进行客户回访和满意度调查

### 1.3 关注阶段
- **S8（现场交付）**：发货、安装、SAT验收
- **S9（质保结项）**：质保期服务与结项

### 1.4 数据范围
- **数据权限**：`customer`（客户级数据可见）

## 三、功能模块详细设计

### 3.1 统计卡片区域

#### 3.1.1 待处理问题
- **指标**：待处理问题总数
- **子指标**：紧急问题数量
- **颜色**：红色系（`from-red-500/10 to-orange-500/5`）
- **图标**：`AlertCircle`
- **数据来源**：问题管理模块，状态为 `OPEN` 或 `IN_PROGRESS` 的问题

#### 3.1.2 进行中服务
- **指标**：进行中的服务任务数
- **子指标**：今日任务数
- **颜色**：蓝色系（`from-blue-500/10 to-cyan-500/5`）
- **图标**：`Wrench`
- **数据来源**：服务任务表，状态为 `IN_PROGRESS` 或 `SCHEDULED` 的任务

#### 3.1.3 本月解决问题
- **指标**：本月已解决的问题数
- **子指标**：环比增长率
- **颜色**：绿色系（`from-emerald-500/10 to-green-500/5`）
- **图标**：`CheckCircle2`
- **数据来源**：问题管理模块，本月状态为 `RESOLVED` 或 `VERIFIED` 的问题

#### 3.1.4 客户满意度
- **指标**：客户满意度评分（5分制）
- **子指标**：环比变化
- **颜色**：紫色系（`from-purple-500/10 to-pink-500/5`）
- **图标**：`Star`
- **数据来源**：客户满意度调查表，最近30天的平均评分

### 3.4 客户问题模块

#### 3.4.1 功能说明
展示分配给当前客服工程师的客户问题列表，支持问题转派功能。

#### 3.4.1.1 问题转派功能
- **转派按钮**：每个待处理问题（状态为`open`）显示"转派"按钮
- **转派对象**：可转派给项目经理、技术工程师、测试工程师等
- **转派说明**：支持填写转派原因和说明
- **完成日期**：可设置要求完成日期
- **转派记录**：转派操作会记录在问题跟进记录中

#### 3.4.2 显示信息
- **问题标题**：问题简要描述
- **问题编号**：唯一标识符（如 `ISS-20260106-001`）
- **关联项目**：问题所属项目
- **客户名称**：提出问题的客户
- **严重程度**：低/中/高/严重（带颜色标识）
- **优先级**：低/中/高/紧急（带颜色标识）
- **问题状态**：待处理/处理中/已解决/已验证/已关闭
- **跟进次数**：问题跟进记录数
- **提出人**：客户联系人
- **提出时间**：问题创建时间
- **要求完成日期**：问题处理截止日期

#### 3.4.3 筛选功能
- **搜索框**：支持按问题标题、项目名称、客户名称、问题编号搜索
- **状态筛选**：待处理、处理中、已解决等
- **优先级筛选**：紧急、高、中、低
- **严重程度筛选**：严重、高、中、低

#### 3.4.4 数据来源
- 问题表（`issues`）：筛选 `assignee_id` 为当前用户的问题
- 关联项目表和客户表获取详细信息

#### 3.4.5 交互功能
- 点击问题卡片跳转到问题详情页
- 支持快速查看跟进记录
- 显示问题处理进度

### 3.8 创建问题工单功能（核心功能）

#### 3.8.1 功能说明
客服工程师的核心功能：能够反馈项目和设备的问题，通过创建问题工单，然后转派给相应的人员。

#### 3.8.2 创建问题工单对话框

**触发方式**：
- 页面头部"创建问题工单"按钮
- 快捷操作中的"创建问题工单"按钮

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 问题分类 | 下拉选择 | 是 | 客户问题/项目问题/任务问题/验收问题/质量问题/技术问题 |
| 问题类型 | 下拉选择 | 是 | 缺陷/偏差/风险/阻塞/建议/疑问 |
| 严重程度 | 下拉选择 | 是 | 低/中/高/严重 |
| 优先级 | 下拉选择 | 是 | 低/中/高/紧急 |
| 关联项目 | 下拉选择 | 否 | 选择关联的项目（可选） |
| 关联设备 | 下拉选择 | 否 | 选择关联的设备（需先选择项目） |
| 问题标题 | 文本输入 | 是 | 问题简要描述 |
| 问题描述 | 多行文本 | 是 | 详细描述问题情况、复现步骤、影响范围等 |
| 转派给 | 下拉选择 | 否 | 选择处理人（项目经理、技术工程师等） |
| 要求完成日期 | 日期选择 | 否 | 设置问题处理截止日期 |
| 影响范围 | 文本输入 | 否 | 描述问题影响范围 |
| 是否阻塞 | 复选框 | 否 | 是否阻塞项目/任务 |

#### 3.8.3 转派功能

**转派对象选择**：
- 项目经理
- 技术工程师（机械/电气/软件/测试）
- 质量工程师
- 其他相关人员

**转派流程**：
1. 客服工程师创建问题工单
2. 填写问题详细信息
3. 选择转派对象（可选，也可创建后不转派）
4. 设置要求完成日期（可选）
5. 提交创建，系统自动：
   - 生成问题编号
   - 记录创建人和创建时间
   - 如果指定了转派对象，自动分配给该人员
   - 发送通知给转派对象
   - 创建问题跟进记录

#### 3.8.4 问题转派对话框

**触发方式**：
- 问题列表中待处理问题的"转派"按钮

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 转派给 | 下拉选择 | 是 | 选择新的处理人 |
| 要求完成日期 | 日期选择 | 否 | 设置新的完成日期 |
| 转派说明 | 多行文本 | 否 | 转派原因或说明 |

**转派操作**：
- 更新问题的处理人
- 更新要求完成日期
- 创建转派跟进记录
- 发送通知给新的处理人
- 如果原处理人存在，通知原处理人问题已转派

#### 3.8.5 数据来源

**项目列表**：
- API: `GET /api/v1/projects?stage=S8,S9`
- 筛选条件：S8（现场交付）和S9（质保结项）阶段的项目
- 显示格式：`项目编码 - 项目名称 (客户名称)`

**设备列表**：
- API: `GET /api/v1/projects/{project_id}/machines`
- 根据选择的项目动态加载
- 显示格式：`设备编码 - 设备名称`

**用户列表**：
- API: `GET /api/v1/users?role=pm,me_engineer,ee_engineer,sw_engineer,te_engineer,qa`
- 筛选可转派的角色
- 显示格式：`姓名 (角色)`

#### 3.8.6 API接口

**创建问题**：
```
POST /api/v1/issues/
Content-Type: application/json

{
  "category": "CUSTOMER",
  "issue_type": "DEFECT",
  "severity": "MEDIUM",

  "title": "设备运行异常",

  "assignee_id": 2,
```

```
  "assignee_id": 3,
  "due_date": "2026-01-12",
```

## 七、权限控制

### 7.1 数据权限
- **数据范围**：`customer`（客户级）
- 只能查看分配给自己的问题和任务
- 只能查看自己参与的项目（S8/S9阶段）

### 7.2 操作权限
- **创建问题**：允许
- **创建服务任务**：允许
- **提交服务报告**：允许
- **创建SAT验收**：允许（需项目经理审批）
- **修改问题状态**：允许（仅自己负责的问题）
- **查看客户信息**：允许（仅自己负责的客户）

# 非标自动化项目管理系统 - 整体规划方案

**编制单位**：深圳金凯博自动化科技有限公司  
**编制日期**：2025年7月  
**版本**：V1.0

---

## 第一部分：现状分析与核心痛点

### 1.1 企业概况
- 员工规模：约200人，其中工程师80+人
- 业务类型：非标自动化测试设备、线体
- 主要客户：汽车电子、白色家电、BMS等行业

### 1.2 核心痛点诊断

| 痛点 | 表现 | 根因分析 | 影响 |
|------|------|----------|------|
| **总是缺料** | 装配时才发现缺料，生产被迫暂停 | ① BOM发布晚 ② 采购跟踪不到位 ③ 设计变更未及时同步 | 交期延误、装配等待 |
| **交期老逾期** | 客户投诉多，信誉受损 | ① 计划不合理 ② 异常响应慢 ③ 缺乏预警机制 | 客户流失、回款困难 |
| **工程师互相等** | 机械等电气、电气等软件、软件等调试 | ① 缺乏协同机制 ② 信息不透明 ③ 资源分配不均 | 效率低下、内耗严重 |
| **设计变更混乱** | 变更频繁，BOM/采购/成本无法联动 | ① 无变更流程 ② 手工通知易遗漏 ③ 版本管理缺失 | 错料、多采购、成本失控 |
| **外协跟踪难** | 外协件是"黑洞"，不知道进度 | ① 缺乏跟踪手段 ② 报表不规范 ③ 责任不清 | 关键件延期 |
| **售后响应慢** | 客户问题微信群里找，无闭环 | ① 无工单系统 ② 问题分散 ③ 无知识沉淀 | 重复问题、客户不满 |
| **项目不复盘** | 做完就完，不知赚没赚，问题重复犯 | ① 无流程要求 ② 数据不完整 ③ 无模板 | 无法改进、成本失控 |

### 1.3 根本原因总结

```
┌─────────────────────────────────────────────────────────┐
│  表面问题：缺料、逾期、等待、变更乱、外协跟不上...        │
├─────────────────────────────────────────────────────────┤
│  管理问题：计划不合理、信息不透明、流程不规范、责任不清    │
├─────────────────────────────────────────────────────────┤
│  底层问题：缺乏统一的数据标准、编码体系、系统支撑         │
└─────────────────────────────────────────────────────────┘
```

### 2.1 主数据标准化

| 数据对象 | 编码规则 | 示例 | 说明 |
|----------|----------|------|------|

| **机台号** | PJxxx-PNxxx | PJ250708001-PN001 | 项目号-PN+3位序号 |

| **供应商编码** | Vxxxxx | V00001 | V+5位流水号 |

| **工程师编码** | Exxxx | E0001 | E+4位工号 |
| **设计变更号** | ECN-PJxxx-xx | ECN-PJ250708001-01 | ECN-项目号-序号 |
| **外协单号** | OT-yymmdd-xxx | OT-250710-001 | OT+年月日+序号 |
| **售后工单号** | SR-yymmdd-xxx | SR-250715-001 | SR+年月日+序号 |

#### 2.1.2 物料编码体系（重点）

```
物料编码 = 大类(2位) + 中类(2位) + 小类(2位) + 规格(4位) + 流水(4位)

大类：
  ME - 机械件     EL - 电气件     PN - 气动件
  ST - 标准件     OT - 外协件     TR - 贸易件

示例：
  ME-01-01-0001-0001 = 机械件-机加件-轴类-规格1-流水1
  EL-02-03-0015-0001 = 电气件-传感器-光电开关-规格15-流水1
```

#### 2.1.3 状态与阶段标准化

**三维状态体系**（已设计，此处确认）：

| 维度 | 代码 | 数量 | 用途 |
|------|------|------|------|
| 阶段 Stage | S1-S9 | 9个 | 里程碑管理 |
| 流程状态 Status | ST01-ST99 | 32个 | 任务协作 |
| 健康度 Health | H1-H4 | 4个 | 预警决策 |

## 第五部分：技术选型建议

### 5.1 技术方案对比

| 方案 | 优点 | 缺点 | 适用场景 | 成本 |
|------|------|------|----------|------|
| **Excel + VBA** | 零成本、快速 | 协同差、数据量受限 | 临时、小规模 | 0 |
| **低代码平台**<br>(简道云/宜搭) | 快速搭建、成本低 | 定制受限、复杂逻辑难 | 中小企业、快速验证 | 1-5万/年 |
| **n8n + 数据库** | 灵活、可扩展、自动化强 | 需要技术能力 | 有IT能力、重自动化 | 服务器成本 |
| **专业ERP**<br>(金蝶/用友) | 功能全、合规 | 重、贵、定制难 | 大企业、上市要求 | 10-50万+ |
| **自研系统** | 完全定制 | 周期长、成本高 | 特殊需求 | 20万+ |

### 5.2 推荐方案

基于金凯博的情况，建议采用**渐进式路径**：

```
当前阶段               短期(6个月)              中期(12个月)
┌─────────┐          ┌─────────────┐          ┌─────────────┐
│  Excel  │   ───▶   │   Excel +   │   ───▶   │  低代码平台  │
│  手工   │          │   n8n自动化  │          │  或自研系统  │
└─────────┘          └─────────────┘          └─────────────┘
     │                      │                        │
     ▼                      ▼                        ▼
 现状：分散           过渡：自动化提醒          目标：完整系统
                         报表生成
                         数据同步
```

**第一步（立即可做）**：
- 用这次设计的Excel模板规范数据
- 用n8n实现自动化提醒（缺料预警、收款提醒）
- 用企业微信/钉钉推送消息

**第二步（验证需求后）**：
- 根据实际使用情况，决定上低代码平台还是自研
